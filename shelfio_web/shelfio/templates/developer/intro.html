Common Pieces

The Shelf.io API allows developers to interact with Shelf.io without using the Shelf.io web interface. Have a look at some of the common API pieces below and reference the Item, Shelf, User, Authorizations, and Favorites for details.

The latest (and only) version of the API is version one, or V1. Its base URL is http://api.shelf.io/v1/. It is a REST-like API. It responds to HTTP verbs; GET, PUT, PATCH, POST, DELETE, and responds in JSON.

Use an auth token

Some resources in Shelf.io are protected in order to provide some privacy. To access these protected resources, you'll need to supply an OAuth token. You can supply the token by placing it in the HTTP header or by tacking it on the parameter list:

In HTTP header

curl -v --header "Authorization: Token 63fd57ca-your-auth-token-c42c033233c6" "http://api.shelf.io/v1/path/to/your/resource/"

In parameter list

curl -v "http://api.shelf.io/v1/path/to/your/resource/?access_token=63fd57ca-your-auth-token-c42c033233c6"

Use HTTPS

Both http://api.shelf.io/v1/ and https://api.shelf.io/v1/ are available. You should use https when possible. You don't want to just put your token out in the open.

JSONP Support

JSONP is a technique that can be used to sidestep the Same Origin Policy found in most browsers (link to http://en.wikipedia.org/wiki/Same_origin_policy). To use the JSONP technique, simply append a parameter called callback, with a value, to your request.

https://api.shelf.io/v1/path/to/your/resource/?callback=foo foo({ "your_response": {"wrapped": "up in a method call"} })
